4. Network activity

The application performs network requests only in the following specific, user-initiated cases.

4.1 Model download (one-time, first launch)

On first launch, after accepting the clinical disclaimer, the application downloads approximately 7.5 GB of AI model files from the Hugging Face Content Delivery Network (huggingface.co). These are downloaded once and stored locally.

  • Sent: standard HTTP GET requests for specific, publicly-available model files.
  • Not sent: any user-generated content, device identifiers, or personal data.

4.2 Knowledge → "Deep" mode (opt-in, per-query)

When the user explicitly sets Knowledge search to Deep mode, individual clinical queries trigger calls to the U.S. National Center for Biotechnology Information (NCBI):

  • NCBI E-utilities API (eutils.ncbi.nlm.nih.gov) for PubMed search
  • BioC-PubMed (www.ncbi.nlm.nih.gov/research/bionlp/) for abstract fetch
  • Sent: the user's query string.
  • Not sent: any prior chat history, drafted notes, attached images, patient information, or device identifiers.

The user can disable Deep mode at any time in Settings to prevent any network activity beyond the initial model download.

4.3 External links

Tapping "Join our Discord" in the sidebar opens the system browser with a Discord URL. This is a standard external link and is not initiated or mediated by the application's own networking.

5. Protected Health Information (PHI)

The application includes a text anonymization step that removes common personally-identifiable patterns (emails, phone numbers, US social security numbers) from user-entered text before sending to the on-device AI model. This is a best-effort feature, not a regulated de-identification tool.

Limitations users must understand:
  • The anonymizer operates on text only. Images attached to messages are not redacted; patient names, faces, or identifiers visible on an image remain visible.
  • The anonymizer uses pattern matching, not natural-language understanding. It may miss non-standard identifiers.
  • Even after anonymization, text is stored in the local Isar database with Android's standard app-private permissions, not encrypted at rest.

Users are solely responsible for deciding whether any information they enter is appropriate for the application. The application is not a HIPAA-compliant, GDPR-compliant, or otherwise regulated medical-data-processing tool.

6. Children

The application is intended for medical-education users aged 18 or older. We do not knowingly collect information from children, because we do not collect information from anyone.

8. Data retention

All application data is retained locally on the device for as long as the app is installed. Uninstalling deletes:

  • All chat history and drafted notes
  • All Analyze history and entity-extraction results
  • All attached image thumbnails
  • All downloaded AI model files
  • All user preferences

We retain no copies of user data because we receive no user data.

9. Data security

The application uses standard Android app-sandboxing. Data is stored in the app's private storage directory, accessible only to this application (and to the device owner with root access).

Network requests (model download, Deep Search) use HTTPS exclusively. The app-level network security configuration blocks cleartext HTTP traffic.

10. Your rights

Because we do not receive, store, or process any personal data on our servers (we have none), most "data subject rights" (access, rectification, deletion, portability) are satisfied automatically by Android's app-uninstall mechanism.

If you wish to be certain no app-held data remains on your device, uninstall the application. Android will delete all app-private storage.
